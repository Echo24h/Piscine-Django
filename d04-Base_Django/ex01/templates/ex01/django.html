Ex01 : Django, framework web

Qu'est-ce que Django ?

Django est un framework web haut niveau écrit en Python, conçu pour aider les développeurs à créer des applications web rapidement et avec un minimum de code. Django suit le principe "Don't Repeat Yourself" (DRY), ce qui signifie qu'il encourage la réutilisation du code et la réduction des redondances. Le framework offre une multitude de fonctionnalités intégrées, telles que :

  • Un ORM (Object-Relational Mapping) pour interagir avec les bases de données.
  • Un système de templating pour générer des pages HTML de manière dynamique.
  • Un système d'authentification robuste.
  • Des outils de validation et de sécurisation des formulaires.
  • Une interface d'administration générée automatiquement.

Django est également reconnu pour sa sécurité, car il inclut des protections contre des attaques courantes comme l'injection SQL, les falsifications de requêtes intersites (CSRF), et le scriptage intersite (XSS).

Historique de Django

  • 2003 : Le développement de Django commence au sein de la société de développement web Lawrence Journal-World, où Adrian Holovaty et Simon Willison travaillent sur plusieurs applications web pour le journal en ligne.
  • Juillet 2005 : Django est publié en open source sous une licence BSD. Le framework est nommé d'après Django Reinhardt, un célèbre guitariste de jazz.
  • Septembre 2008 : Django 1.0 est officiellement publié. Cette version marque la maturité du framework et introduit une API stable.
  • Mars 2011 : Django 1.3 est publié, apportant le support pour les applications réutilisables et le déploiement simplifié.
  • Octobre 2013 : Django 1.6 introduit le support pour les migrations de bases de données via l'outil South.
  • Septembre 2014 : Django 1.7 intègre officiellement les migrations de bases de données, éliminant la nécessité de South.
  • Avril 2015 : Django 1.8 est publié et devient la première version à bénéficier d'un support à long terme (LTS).
  • Août 2016 : Django 1.10 introduit les middlewares, améliorant la modularité et la gestion des requêtes/réponses.
  • Avril 2019 : Django 2.2 est publié, offrant des améliorations de performance et une meilleure compatibilité avec Python 3.7.
  • Décembre 2020 : Django 3.1 apporte le support pour les fonctionnalités asynchrones, permettant de meilleures performances pour les applications web modernes.
  • Avril 2021 : Django 3.2 est publié en tant que version LTS, intégrant de nombreuses améliorations et de nouvelles fonctionnalités.

Aujourd'hui, Django est utilisé par de nombreuses grandes entreprises et organisations, telles que Instagram, Pinterest, Mozilla, et la NASA. Sa communauté active et sa documentation exhaustive en font un choix populaire pour les développeurs souhaitant créer des applications web robustes et évolutives.
