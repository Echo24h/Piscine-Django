Ex01 : Processus d'affichage d'une page statique

Introduction

Ce document décrit le processus complet de l'affichage d'une page web statique dans un projet Django, depuis la requête initiale du client jusqu'à la réponse du serveur, en passant par une vue et l'utilisation d'un template.

Étapes du processus

1. La requête HTTP

Tout commence lorsqu'un utilisateur saisit une URL dans son navigateur ou clique sur un lien. Le navigateur envoie alors une requête HTTP vers le serveur Django.

2. Le routeur URL

À la réception de la requête, Django utilise son système de routage d'URL pour déterminer quelle vue doit traiter la requête. Ce processus est basé sur les correspondances entre les URL définies dans urls.py et les vues associées.


# urls.py
from django.urls import path
from . import views

urlpatterns = [
path('affichage/', views.affichage, name='affichage'),
]

3. La vue

Une fois que le routeur URL a trouvé une correspondance, il appelle la vue correspondante. La vue est une fonction ou une classe qui contient la logique nécessaire pour traiter la requête et préparer une réponse. Pour une page statique simple, la vue peut se contenter de rendre un template.


# views.py
from django.shortcuts import render

def affichage(request):
return render(request, 'ex01/affichage.html')

4. Le moteur de template

La vue appelle le moteur de template de Django pour rendre le fichier HTML spécifié. Le moteur de template charge le template, remplit les variables dynamiques (s'il y en a) et produit le contenu HTML final.

5. La réponse HTTP

Le contenu HTML généré par le moteur de template est encapsulé dans une réponse HTTP. Django envoie cette réponse au navigateur de l'utilisateur.

6. L'affichage dans le navigateur

Le navigateur reçoit la réponse HTTP et affiche le contenu HTML à l'utilisateur, complétant ainsi le cycle de la requête à la réponse.

Résumé

Le processus d'affichage d'une page web statique dans Django suit un chemin bien défini :

  1. Le navigateur envoie une requête HTTP au serveur Django.
  2. Django utilise le routeur URL pour faire correspondre la requête à une vue.
  3. La vue traite la requête et utilise le moteur de template pour rendre un fichier HTML.
  4. Django envoie la réponse HTTP contenant le HTML rendu au navigateur.
  5. Le navigateur affiche la page web statique à l'utilisateur.

Ce processus est simple mais puissant, permettant une grande flexibilité dans la manière dont les pages web sont générées et affichées.
