Ex01 : Moteur de template

Introduction

Le moteur de templates de Django permet de générer du HTML de manière dynamique en séparant la logique de l'affichage. Les templates peuvent inclure des variables et des structures de contrôle pour créer des pages web dynamiques et réutilisables.

Fonctionnement du moteur de templates

Le moteur de templates de Django utilise des balises et des filtres pour insérer du contenu dynamique dans les pages HTML. Les templates sont des fichiers texte contenant des balises spéciales entourées de crochets {% %} pour les balises de contrôle et de doubles crochets {{ }} pour les variables.

Les blocs

Les blocs permettent de définir des sections réutilisables et remplaçables dans les templates. Ils sont souvent utilisés dans les templates de base pour permettre l'héritage des templates enfants.


{% raw %}{% block content %}

Contenu par défaut. Remplacé dans les templates enfants.

{% endblock %}{% endraw %}

Les boucles for ... in

Les boucles for ... in sont utilisées pour itérer sur des listes ou des objets semblables à des listes. Elles permettent de générer du contenu répété dans les templates.


{% raw %}{% for item in items %}

{{ item }}

{% endfor %}{% endraw %}

Les structures de contrôle if

Les structures de contrôle if permettent d'inclure des conditions dans les templates. Elles peuvent être utilisées pour afficher ou masquer du contenu en fonction de conditions spécifiques.


{% raw %}{% if user.is_authenticated %}

Bonjour, {{ user.username }} !

{% else %}

Bonjour, visiteur !

{% endif %}{% endraw %}

Affichage de variables passées par contexte

Les variables passées par contexte sont insérées dans les templates en utilisant des doubles crochets {{ }}. Ces variables sont souvent fournies par les vues et peuvent contenir des données dynamiques.


{% raw %}
{{ message }}
{% endraw %}
